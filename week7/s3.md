# Amazon S3 (Simple Storage Service)

# Introduction to Amazon S3

Amazon S3 (Simple Storage Service) is an object storage service provided by AWS.

It is used to store and retrieve:

* Files
* Images
* Videos
* Backups
* Logs
* Static websites
* Application assets

S3 provides highly scalable, durable, and secure cloud storage.

---

# Key Features of S3

* Unlimited storage scalability
* High durability
* High availability
* Secure access control
* Versioning support
* Lifecycle management
* Static website hosting
* Cross-region replication

---

# S3 Architecture

```text id="f4k9mz"
User
  ↓
S3 Bucket
  ↓
Objects (Files)
```

---

# Important S3 Terms

| Term          | Description                        |
| ------------- | ---------------------------------- |
| Bucket        | Container for storing objects      |
| Object        | Actual file/data stored            |
| Key           | Unique object name                 |
| Versioning    | Multiple versions of objects       |
| Storage Class | Storage tier for cost optimization |

# S3 Bucket Naming Rules

* Must be globally unique
* Only lowercase letters allowed
* Cannot contain spaces
* Supports numbers and hyphens

### Example

```text id="m3q8xd"

```

# S3 Storage Classes (Storage Tiers)

S3 provides multiple storage classes based on access frequency and cost optimization.

---

# 1. S3 Standard

Used for frequently accessed data.

### Features

* High availability
* Low latency
* Multi-AZ storage

### Use Cases

* Websites
* Applications
* Frequently accessed files

---

# 2. S3 Intelligent-Tiering

Automatically moves objects between tiers based on usage.

### Benefits

* Cost optimization
* Automatic tier management

---

# 3. S3 Standard-IA

IA = Infrequent Access

### Features

* Lower storage cost
* Retrieval charges apply

### Use Cases

* Backups
* Disaster recovery

---

# 4. S3 One Zone-IA

Stores data in a single Availability Zone.

### Benefits

* Lower cost

### Limitation

* Lower durability compared to multi-AZ

---

# 5. S3 Glacier Instant Retrieval

Used for archival data with quick retrieval.

### Use Cases

* Medical records
* Archived files

---

# 6. S3 Glacier Flexible Retrieval

Low-cost archival storage.

### Retrieval Time

Minutes to hours

---

# 7. S3 Glacier Deep Archive

Lowest-cost archival storage.

### Retrieval Time

Up to 12 hours

---

# Storage Class Comparison

| Storage Class       | Access Frequency | Cost      |
| ------------------- | ---------------- | --------- |
| Standard            | Frequent         | High      |
| Intelligent-Tiering | Variable         | Optimized |
| Standard-IA         | Infrequent       | Lower     |
| Glacier             | Rare             | Very Low  |

# IAM Policies

IAM policies define permissions for users and roles.

### Example

```json id="x6j4rt"
{
  "Effect": "Allow",
  "Action": "s3:*",
  "Resource": "*"
}
```

---

# Bucket Policies

Bucket policies are resource-based policies attached directly to S3 buckets.

---

# Example Bucket Policy

```json id="v2p9ya"
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*"
    }
  ]
}
```

---

# ACL (Access Control List)

ACLs provide object-level and bucket-level permissions.

### Permissions

* Read
* Write
* Full Control

---

# Bucket Policy vs ACL

| Bucket Policy        | ACL                    |
| -------------------- | ---------------------- |
| JSON based           | Simple permission list |
| More flexible        | Limited permissions    |
| Recommended by AWS   | Older access model     |
| Bucket-level control | Object-level control   |

---

# Block Public Access

AWS provides Block Public Access settings to prevent accidental public exposure.

### Recommended

Always enable unless hosting public content.

# Terraform for S3

Terraform can automate S3 bucket creation and management.

---

# Terraform S3 Bucket Example

```hcl id="h7q3wb"
resource "aws_s3_bucket" "demo" {
  bucket = "vishnu-demo-bucket"

  tags = {
    Name = "DemoBucket"
  }
}
```

---

# Enable Versioning Using Terraform

```hcl id="d2k8zc"
resource "aws_s3_bucket_versioning" "versioning" {
  bucket = aws_s3_bucket.demo.id

  versioning_configuration {
    status = "Enabled"
  }
}
```

---

# Static Website Hosting Using Terraform

```hcl id="g5w1rn"
resource "aws_s3_bucket_website_configuration" "website" {
  bucket = aws_s3_bucket.demo.id

  index_document {
    suffix = "index.html"
  }
}
```

---

# Bucket Policy Using Terraform

```hcl id="k9n2pd"
resource "aws_s3_bucket_policy" "public_policy" {
  bucket = aws_s3_bucket.demo.id

  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [{
      Effect = "Allow"
      Principal = "*"
      Action = "s3:GetObject"
      Resource = "${aws_s3_bucket.demo.arn}/*"
    }]
  })
}
```
